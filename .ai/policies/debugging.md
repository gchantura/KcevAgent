# Root-cause debugging policy

1. Do not guess.
2. Read the exact error message and preserve the relevant evidence.
3. Identify the exact file and line when available.
4. Explain the cause briefly.
5. Fix only the root cause.
6. Re-run the failing validation and relevant regression checks.
7. Do not rewrite unrelated code.

If evidence is insufficient, gather more diagnostics instead of editing speculatively.
