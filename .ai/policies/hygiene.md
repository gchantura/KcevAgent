# Repository hygiene policy

- Remove temporary files, scripts, fixtures, debug output, screenshots, generated experiments, and code used only to verify a hypothesis before completing the task.
- Remove temporary dependencies and configuration after use.
- Keep permanent regression tests when they protect intended behavior; tests are not temporary artifacts.
- Do not delete user-owned or pre-existing files merely because they appear unused. Verify ownership and references first.
- Leave the project structure cleaner, minimal, and easy to understand. Do not leave commented-out code or unexplained duplicates.
