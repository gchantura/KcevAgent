# UI quality policy

- Make every page responsive on mobile, tablet, and desktop without breaking the existing layout.
- Avoid repeated hardcoded styles. Reuse existing design tokens, utilities, and components.
- Keep text readable and controls visible, keyboard-accessible, and appropriately labelled.
- Give every form control an accessible label and useful validation feedback.
- Provide loading and error states for asynchronous behavior.
- Provide empty states when absence of data is a normal condition.
- Check overflow, focus visibility, touch targets, content reflow, and meaningful semantic structure.
